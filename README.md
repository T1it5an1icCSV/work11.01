# work11.01 — учебный проект (TechLine)

Учебный проект по курсу проектирования, разработки и документирования защищённой клиент‑серверной информационной системы на базе СУБД.

## Неделя 1. Постановка задачи и выбор предметной области

- Выбор предметной области (интернет‑магазин «TechLine»).
- Формулировка целей и задач проекта.
- Определение основных входных и выходных данных.
- Начата общая часть пояснительной записки (описание предметной области и ролей пользователей).

## Неделя 2. Проектирование базы данных

- Построена ER‑диаграмма предметной области.
- Определены сущности и связи (1:1, 1:M, M:M).
- Выполнена нормализация до 3‑й нормальной формы.
- Подготовлена специальная часть пояснительной записки, посвящённая структуре БД.

## Неделя 3. Реализация базы данных

- В СУБД созданы таблицы и связи в соответствии с проектом.
- Определены ограничения целостности (PRIMARY KEY, FOREIGN KEY, UNIQUE, CHECK, NOT NULL).
- Добавлены тестовые данные для основных сущностей.
- Составлена документация: структурная схема БД и словарь данных.

## Неделя 4. Серверная логика: базовые функции

- Реализованы CRUD‑операции для ключевых сущностей (товары, категории, заказы, пользователи и т.д.).
- Созданы представления (VIEW) для удобного чтения и отчётности.
- Написаны первые хранимые процедуры.
- Листинги кода включены в приложения к пояснительной записке.

## Неделя 5. Расширенная серверная логика и защита

- Реализовано не менее трёх хранимых процедур, функций и триггеров.
- Добавлены транзакции с поддержкой откатов для бизнес‑операций.
- Выполнена серверная валидация данных (ограничения в БД, проверки в коде).
- Начата глава «Механизмы защиты» в специальной части пояснительной записки.

## Неделя 6. Безопасность и администрирование

- Настроено разделение прав доступа (RBAC, минимум три уровня ролей: гость/пользователь/администратор/менеджер).
- Пароли хранятся в виде безопасного хэша.
- Настроены сценарии резервного копирования и восстановления БД.
- Ведётся логирование действий пользователей (журнал аудита).
- Документированы меры защиты и администрирования в пояснительной записке.

## Неделя 7. Клиент‑серверное приложение и API

- Разработан клиентский интерфейс (веб‑приложение на основе шаблонов).
- Реализован RESTful API для основных операций.
- Выделено серверное ядро и дополнительные сервисы (при необходимости — микросервисный подход).
- Доработана документация: схемы пользовательских интерфейсов и архитектуры сервиса.

## Неделя 8. Подсистема статистики и отчётности

- Реализована подсистема статистики и аналитики на основе представлений и запросов.
- Подготовлены графики и визуализации основных показателей.
- Оформлен раздел «Результаты работы программы» в пояснительной записке.

## Неделя 9. Тестирование

- Проведено функциональное, интеграционное и нагрузочное тестирование.
- Выполнен анализ отказоустойчивости и тестирования безопасности.
- Частично автоматизированы unit‑тесты (SQL‑логика и/или API).
- Оформлены тест‑планы, тест‑кейсы и отчёты о результатах прогонов.

## Неделя 10. Документация: технологическая часть

- Описаны используемые языки, фреймворки, СУБД и инструменты разработки.
- Задокументированы процессы отладки и меры защитного программирования.
- Представлена DevOps‑схема (Git, CI/CD, при необходимости — контейнеризация).
- Описаны основные характеристики системы: производительность, безопасность, эксплуатационные особенности.

---

## Запуск проекта (кратко)

1. Установить Java и PostgreSQL.
2. Создать базу данных и выполнить начальные скрипты из папки `src/main/resources`.
3. Настроить подключение к БД в `application.properties`.
4. Собрать и запустить Spring Boot приложение.
5. Открыть веб‑интерфейс в браузере (`http://localhost:8081` или другой сконфигурированный порт).

Проект предназначен для демонстрации профессиональных компетенций по проектированию, реализации и администрированию защищённых информационных систем.
